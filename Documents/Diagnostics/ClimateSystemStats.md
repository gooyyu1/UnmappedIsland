# 気候システム統計レポート

本ドキュメントは、現在の気候システム実装（`ClimateSystem.md`、`Assets/StreamingAssets/WorldCodex/core.yaml`）
について、複数シードのシミュレーションで実測した統計値のスナップショットです。
`Tests/Diagnostics/ClimateStatsReportTests.cs`（`[Explicit]`、通常のテスト実行には含まれない）を
以下のコマンドで実行すると再生成されます。

```
dotnet test Tests/Tests.csproj --filter FullyQualifiedName~ClimateStatsReportTests
```

**このレポートはスナップショットであり、確定仕様ではありません。** レート・閾値等の
バランス調整で数値は変わりうるため、`core.yaml`を変更した後は再生成してください。

## 計測方法

- シード数: 20 / 1シードあたりの実測期間: 3600日
- `WorldSession.AdvanceWorldTime`経由でtickを進めており、hour/dayも実際のゲーム同様に進行する
  （そのため気温には日照による日内変動も反映されている）。
- 各シードの最初のセグメント（開始点が不明瞭）と末尾の未完了セグメントは統計から除外。
- 「序盤/中盤/終盤」は、各季節インスタンス（毎回の実際の持続日数）を3等分した区間。
- 天気ごとの発生時間は「その期間の間にその天気であった合計時間」。標本単位は季節インスタンス
  （序盤/中盤/終盤はその3等分区間）で、一度も発生しなかった期間は**0時間の標本として計上**する。
  そのためnは全天気で共通（=完了した季節インスタンス数）になる。
- 連続降雨/未降雨時間は「同じ状態が連続した1回ごとの長さ」を標本とする（発生時間とは考え方が異なる）。
  連続区間は、その**開始tick**が属する序盤/中盤/終盤に割り当てる（季節の境界をまたぐ区間は
  境界で打ち切り、両側で計上しない）。
- 標準偏差は標本標準偏差（n-1）。

同じシミュレーションのtickごとの変化量から実測して求めている（バランス調整のたびに
転記し直す手間と、転記漏れによる`core.yaml`との乖離を避けるため）。前後どちらかのtickで
大気水分量が範囲端（0/10,000）に達しているサンプルはクランプにより真の値より小さく
