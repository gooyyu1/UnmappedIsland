    FixedPositions

    前詰めしないか（既定false）。trueの場合、Runtime側（Slot）が「型→固定番号」の対応表を持ち、 空いた番号を保持したまま詰めない・プレイヤーが手動で並び替え可能、という挙動になる （例: プレイヤー手持ちの6枠）。

    Stackable

    同種オブジェクトを表示上1つの単位（スタック）としてまとめるか（既定true）。falseなら同種でも 個体ごとに別単位として数える（例: かまどの投入口。同じ種類の燃料を2つ入れても2枠消費する）。

    UnitCapacity

    このスロットに同時に存在できる「単位」の上限（null=無制限）。単位の意味はStackableに従う （trueなら異なるObjectDefの種類数、falseなら個体数そのもの）。既存のCapacity（サイズ合計） とは独立した、種類数/個数ベースの別軸の制約。
