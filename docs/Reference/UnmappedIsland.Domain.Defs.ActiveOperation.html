    Class ActiveOperation

    一時的な命令(active)の1操作（set/add/destroy/spawn/transfer、9節）。owner（この操作を発する自分自身）の 文脈（actor/dragged）を受け取り、「対象を解決して自分を適用する」までを自分で行う（PropertyValueへの 登録で持続するpassiveのPassiveEffectと対になる、一度きり側のポリモーフィズム。自分のことは自分でする、 CLAUDE.md参照）。

    contextは1回の適用（1つのActiveEffect.ApplyTo）で共有される。same_slot spawnが必要とする「destroyで 失われる前のselfの位置」を、destroy側とspawn側が同じcontextを通して受け渡すために使う（それ以外の 操作は無視してよい）。
