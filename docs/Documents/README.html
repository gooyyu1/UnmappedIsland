UnmappedIsland ドキュメント

このディレクトリにはゲーム開発に関するすべてのドキュメントが格納されています。

ディレクトリ構成

企画・設計・仕様という工程でフォルダを分けると、検討が進むにつれて「これはもう設計と呼べるのでは」 「仕様として確定したのでは」という判断のたびにファイルを移動する必要が生じ、境界があいまいになります （実際、ほとんどのドキュメントは冒頭に「本ドキュメントは検討結果であり、確定仕様書ではありません」と 書いており、工程による分類がうまく機能していませんでした）。

代わりに、扱っている領域でフォルダを分けています。領域は検討の進み具合によらず変わらないため、 ドキュメントの成熟度が上がっても別フォルダへ移す必要がありません。

Documents/
├── Concept/       # コンセプト（企画の核となる考え方。ターゲット・マネタイズ等を含む）
├── UI/            # UI/UX（画面レイアウト、モック、スクリーンショット）
├── Engine/        # ルール・エンジン（WorldCodexのYAML文法そのもの、汎用サブシステム、
│                  #   C#実装のアーキテクチャ・コード構造）
├── World/         # 実際にゲームに登場する内容（地形・気候・アイテムなど、文法を使って
│                  #   表現された具体的なゲーム内容）
└── Diagnostics/   # 診断・統計レポート（実装の挙動を計測したスナップショット。設計判断ではなく測定値）

各フォルダの用途

フォルダ 内容
Concept/ ゲームコンセプト、目指す体験、ターゲット、マネタイズ計画など
UI/ 画面レイアウト、モック、UI/UX設計
Engine/ WorldCodex（YAML）の文法リファレンス、汎用サブシステム（レシピ・コンテナ・耐久値・アクション使い分け）の設計、C#実装のアーキテクチャ・コード構造ガイド
World/ 気候、地形生成、探索・道、アイテムなど、実際にこのゲームに登場する具体的な内容
Diagnostics/ 実装をシミュレーションして計測した統計レポート（Tests/Diagnostics/の[Explicit]テストで再生成する測定値のスナップショット。設計そのものではない）

Diagnostics/ は他の4フォルダと性質が異なります。他は「こう設計する」という判断を記すドキュメントで あるのに対し、Diagnostics/ は「今の実装を測るとこうなった」という結果であり、実装を変えるたびに 手で書き換えるのではなく再生成します（詳細は Diagnostics/README.md）。

備考: ユーザー向けのゲーム内ヘルプやチュートリアルはゲームコンテンツとして実装するため、このフォルダには含まれません。
